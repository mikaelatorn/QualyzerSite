Release 1.2.1 - 12 February 2012

This is a minor release that fixes a problem that occurred where accented characters were included in a coded fragment.

For a detailed list of changes, please consult the issues for milestone 1.2,1.

Release 1.2 - 7 December 2011

This is a major release featuring a new (and more robust) transcript editor, new usability features, and bug fixes.

  • Qualyzer is now able to import a wider variety of RTF files generated by different text editors. In particular, RTF files with images can be loaded without problems (the images will be removed).
  • Accented characters, such as "é", can now be used in codes and participant ids.
  • HTML characters in fragments will no longer cause problems.
  • Users opening Qualyzer for the first time will see a "Welcome Page" guiding them to the main features.

For a detailed list of changes, please consult the issues for milestone 1.2.

Release 1.1.2 - 22 December 2010

This is a minor release with many small bug fixes and improvements. Thanks to Greg Bryant and RJ Leamaster for their help in finding and fixing these various bugs.

  • Improved the RTF file import process: RTF files produced by most office programs should now be imported correctly.
  • Significantly improved the transcript and memo editor performance: large files that used to take 10 seconds to save and open will now take less than a second.
  • Fixed Qualyzer for Mac OSX 64 bit. A missing file was preventing Mac users from opening the transcript and the memo editors.
  • Fixed a bug that prevented viewing fragments associated with a code.
  • Added the Import/Export Project actions to the main File menu.
  • Added an option to export all codes with their associated fragments in a csv file that can be opened in Excel. This feature can be accessed by right-clicking on "Codes" and then selecting "Export | Codes (with Fragments)".

Release 1.1.1 - 3 October 2010

This is a minor release:

  • Added a feature to ease the reporting of issues to Qualyzer developers. To access this feature, go the the Help menu and select Report Issue .

Release 1.1 - 16 September 2010

In addition to all of the Qualyzer 1.0 features, this release includes:

  • Support for code hierarchies. Codes can now be organized in hierarchies in the Code Editor.
  • The ability to associate timestamps with lines in a transcript. Clicking on a timestamp will seek the interview audio file at the recorded time.
  • The option to select the font used in the transcript and memo editors.
  • Support for automatic update of projects created in a previous version of Qualyzer starting from 1.0.
  • Many bug fixes: Qualyzer is more robust than ever!

For a detailed list of changes, please consult the issues for milestone 1.1.

Release 1.0 -23 August 2010

This is the first official release of Qualyzer. Projects created with this release will be supported in future versions. To backup data stored in Qualyzer, simply export your projects into a zip file. Codes can also be saved in a CSV file readable by Microsoft Excel.

In addition to all of the Qualyzer 0.3.0 features, this release includes:

  • Support for accented characters (unicode);
  • Support for importing and exporting projects;
  • The ability to export a list of codes as CSV (excel-compatible format);
  • Customizable shortcut keys to seek within audio files.
  • The option to rename projects.
  • Support for white spaces in project, investigator, participant, transcript, memo, and code names.

For a detailed list of changes, please consult the issues for milestone 1.0.

Release 0.3 -3 August 2010

Important This is an alpha release and it should not be used for a real project: bugs, data loss, and incompatibilities with future releases may happen. This is a list of the main changes in this release:

  • Added Create/Edit/Delete Memos.
  • Added Style Bar to Transcript and Memo editors.
  • Added Audio Bar to Transcript editor.
  • Added the ability to search for fragments associated with a code.
  • Added Code Frequencies (number of associated fragments per code) in Code editor.
  • Created a native Windows installer program.

For a detailed list of changes, please consult the issues for milestone 0.3.

Known Limitations

  • If an audio file is playing when the Transcript editor is closed, it will keep playing until Qualyzer is closed or the audio file ends.

Release 0.2 -14 July 2010

Important This is an alpha release and it should not be used for a real project: bugs, data loss, and incompatibilities with future releases may happen. This is a list of the main changes in this release:

  • Added a Rich Text Format editor (RTF) allowing bold, italic, and underline in transcripts.
  • Added Create/Edit/Delete Code actions.
  • Added the ability to associate a text fragment in a transcript with a code.

For a detailed list of changes, please consult the issues for milestone 0.2.

Known Limitations

  • Pressing F2 causes the program to crash.
  • Entering names (e.g., participant id) with more than 255 characters can corrupt the project. The project then needs to be deleted.
  • RTF files created in Word 2007 must first be opened in WordPad or an earlier version of Word and then "saved as" in order to be successfully imported.
